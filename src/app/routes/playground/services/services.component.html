Services

D.R.Y. - Don't Repeat Yourself. Services are used when you have functionality that you want to implement in multiple components. Instead of including the code in every component you need it in and having to go to each component if you need to change something, you can create a service.

With Components and Directives you create them with a Class Decorator then export the class, but with services it is just a TypeScript class that can be injected (with dependency injection) into multiple components or used as central business logic (like authentication). Services can also help when you need to communicate between multiple components.

After we create/define our service in our .service.ts file we use Angular's dependency injection methods to use our services in our application. The dependency injection injects an instance of our service class into a component we are working in to make it available to the component automatically.

We must first import our class into the component we want to use our service into. Then, to inform Angular that we need to inject the instance of the class, we configure the component's constructor() method. Then finally, we need to provide the service (tell Angular how to create it) by addind the providers: [ ServiceClassName ] property inside the @Component class decorator inside the component.

Hierarchical Injector

When you provide a service somewhere in our app, the Angular framework creates the instance of the service not only in the component where it's specified, but all of it's children components get the same instance of the service.

The highest possible level we can include our service is the AppModule which will make it to where the same instance of a Servidce is available application-wide. When we include our Service in the AppComponet the same instance of the service is available for all COMPONENTS, but not for other Services (depending on where you specify your services).

When you want to have the same instance of a service you include the service in a parent component, you then have access to the same instance of the service in all of the parent component's children.

The way to include the service in a child component and it have the same instance as the parent is to do all the same things, except do not include it in the providers: property of the component. You still import the service and configure it it the component's constructor method.

We can also inject services instances inside other services. You can use this by including the @Injectable() class decorator inside your service which is imported by @angular/core

You include the @Injectable() class decorator in the service that is receiving the other service.

You then configure the service's constructor() method to include the instance from the other service just like we would in a component.
