Component Lifecylce

When an Angular component is initiated, it goes through some lifecycle hooks, that we can hook into and add our own functionality when these hooks are triggered.

  • ngOnChanges -

    Called after a bound input property changes, whenever a bound input properties (like @Input) changes.

  • ngOnInit -

    Called once the component is initilized (has not been added to the DOM yet but Angular is finished with initilization so we can access properties/methods), runs after the constructor.

  • ngDoCheck -

    Called during every change detection run, this hook runs whenever an event is triggered and will check to see whether something has changed within the template so it will be run multiple times.

  • ngAfterContentInit -

    This is called whenever the content that is projected by a ng-content has been initilized.

  • ngAfterContentChecked -

    Called every time the projected content has been checked.

  • ngAfterViewInit -

    Called after the component's view (and child views) has been initialized.

  • ngAfterViewChecked -

    Called every time the view (and child views) has been checked

  • ngOnDestroy -

    Called once a component is destroyed, like if you set *ngIf on one and it is set to false (therefore it removes it from the DOM) ngOnDestroy() is called.
