Directives

Directives are markers on a DOM element that tells the Angular HTML Compiler to attach a specified behavior to that DOM element.

Structural Directive - directives that are began with a * like *ngIf to indicate that this directive directly manipulates the DOM whether it is adding or removing elements or both.

Attribute Directive - directives that look like a normal HTML attribute that only change the element they were placed on.

View the Contents of the basic-directive folder that shows how to create a basic custom directive with the @Directive class decorator.

Like components, we must include directives in our app.module declarations to be used within our app.

Custom Structural Directives

Including the * in structural directives is important because it dynamically generates a < ng-template > tag with the directive that allows you to structure and manipulate the DOM.

ng-template is a template that is not rendered, but allows us to define a template for Angular to decide to use once it determines if the template needs to be rendered.

To create a custom structural directive you start the exact same way as you would with attribute directives except with you start by setting a @Input setter method. Although you set a setter method, you are still declaring a property. It is just a setter of the property which is a method that gets executed whenever the property (the condition we pass) changes.
